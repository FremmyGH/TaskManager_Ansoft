# 📝 Task Manager (WPF MVVM)

Десктопное приложение для управления списком задач, разработанное на платформе **WPF** с использованием архитектурного паттерна **MVVM** и возможностей **C# 12 / .NET 8**.

---

### 🚀 Основные возможности

*   **➕ CRUD задач**  
    Полный цикл управления: добавление, просмотр, редактирование и удаление. Операции проводятся через модальные диалоги с изоляцией данных.

*   **🗑️ Удаление с подтверждением**  
    Защита от случайных действий через встроенные диалоговые окна подтверждения.

*   **🔍 Умный поиск и фильтрация**  
    Поиск по названию/описанию и фильтрация по статусу (Все / Активные / Завершенные) работают в **реальном времени** через `ICollectionView`.

*   **⚠️ Контроль дедлайнов**  
    Автоматическое визуальное выделение просроченных задач (красный фон строки).

*   **🎨 Цветовая индикация приоритетов**  
    *   🔴 **Высокий** — критическая важность.
    *   🟠 **Средний** — стандартные задачи.
    *   🟢 **Низкий** — фоновые дела.

*   **✔️ Строгая валидация (MVVM)**  
    Реализована через `INotifyDataErrorInfo`. Система не позволит сохранить задачу без названия (мин. 3 символа) или со сроком в прошлом.

*   **💾 Асинхронное сохранение JSON**  
    Загрузка при старте и автосохранение изменений в локальный файл `tasks.json` без блокировки интерфейса.

---

### 🛠 Технологический стек

*   **Framework:** .NET 8 (Windows Presentation Foundation).
*   **Архитектура:** MVVM.
*   **Base Logic:** Собственные реализации `BaseViewModel` (валидация), `RelayCommand` и `AsyncRelayCommand`.
*   **Библиотеки:** 
    *   `Microsoft.Xaml.Behaviors.Wpf` — обработка событий в XAML.
    *   `System.Text.Json` — быстрая сериализация данных.
*   **UI Helpers:** `GridHelper` для лаконичной разметки сеток.

---

### 📦 Инструкция по запуску

1.  **Требования:** Установленная **Visual Studio** и компонент **.NET 8 SDK**.
2.  **Клонирование:**
    ```bash
    git clone https://github.com/FremmyGH/TaskManager_Ansoft
    ```
3.  **Сборка:** Откройте файл решения `.sln` и выполните `Build Solution`. Пакеты NuGet восстановятся автоматически.
4.  **Запуск:** Нажмите `F5`. Файл данных `tasks.json` создастся в директории `/bin/Debug/` после добавления первой задачи.

### 📂 Структура проекта

*   **`Model/`** — Сущности данных (`TodoTask`) и перечисления.
*   **`ViewModel/`** — Контексты окон (`TaskManagerContext`, `TaskEditContext`).
*   **`View/`** — XAML разметка интерфейса и стили.
*   **`Model/Lib/`** — Инфраструктурные классы (Команды, Хелперы, Базовые классы).
*   **`Model/Converters/`** — Набор `IValueConverter` для трансформации данных в UI.

---

### 💡 Особенности реализации

*   **Zero Code-Behind:** В проекте принципиально не используется файл `.xaml.cs` для бизнес-логики. Все действия (включая закрытие окон) управляются через команды и триггеры.
*   **StatusBar:** Нижняя панель информирует пользователя о времени последнего сохранения и общем количестве задач в списке.
