📝 Task Manager (WPF MVVM)
Современное настольное приложение для управления списком задач, разработанное на платформе WPF с использованием архитектурного паттерна MVVM.
🚀 Функциональные возможности
CRUD задач: Создание, редактирование и удаление через модальные диалоги.
Умная фильтрация: Поиск по названию/описанию и фильтр по статусу в реальном времени.
Статусы и Приоритеты: Гибкое управление состоянием задач (Active/Completed) и важностью.
Визуальные индикаторы:
⚠️ Автоматическое выделение просроченных задач (розовый фон).
🎨 Цветовая индикация приоритетов (Высокий — красный, Низкий — зеленый).
💬 Информативный StatusBar с уведомлениями о сохранении.
Persistence: Асинхронное сохранение данных в локальный файл tasks.json.
Валидация: Проверка полей (название от 3 символов, дедлайн не в прошлом) через INotifyDataErrorInfo.
🛠 Технологический стек
Язык: C# 12 (.NET 8)
UI: WPF (XAML)
Архитектура: MVVM (Custom BaseViewModel, RelayCommand)
Библиотеки:
Microsoft.Xaml.Behaviors.Wpf (для работы с событиями без Code-Behind).
System.Text.Json (для работы с данными).
Инструменты: Visual Studio 2022.
📦 Инструкция по запуску
1. Требования
Установленный .NET 8 SDK.
Visual Studio 2022 (с рабочей нагрузкой ".NET Desktop Development").
2. Клонирование и запуск
Склонируйте репозиторий:
bash
git clone [ССЫЛКА_НА_ТВОЙ_РЕПОЗИТОРИЙ]
Используйте код с осторожностью.

Откройте файл решения .sln в Visual Studio.
Дождитесь восстановления NuGet-пакетов (произойдет автоматически).
Нажмите F5 или кнопку Start для запуска.
3. Где хранятся данные?
После первого создания задачи в папке с исполняемым файлом (/bin/Debug/net8.0-windows/) появится файл tasks.json. Его можно просматривать и редактировать любым текстовым редактором.
📂 Структура проекта
Model — сущности данных (TodoTask) и перечисления.
ViewModel — логика интерфейса (TaskManagerContext, TaskEditContext).
View — XAML-описания окон.
Services — логика работы с JSON (TodoDataService).
Helpers — кастомные RelayCommand, GridHelper и базовые классы.
Converters — логика преобразования данных для UI.
