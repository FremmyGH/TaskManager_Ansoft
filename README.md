📝 Task Manager (WPF MVVM)
Современное настольное приложение для управления списком задач, разработанное на платформе WPF с использованием архитектурного паттерна MVVM и C# 12.
🚀 Основные возможности
➕ CRUD задач
Добавление и редактирование реализовано в отдельном модальном диалоге с полной изоляцией данных до момента сохранения.
🗑️ Удаление с подтверждением
Защита от случайных действий через диалоговое окно подтверждения.
🔍 Умный поиск и фильтрация
Поиск по названию/описанию и фильтр по статусу (Все/Активные/Завершенные) работают в реальном времени.
⚠️ Контроль дедлайнов
Визуальное выделение просроченных задач (нежно-красный фон строки).
🎨 Цветовая индикация
Приоритеты задач подсвечиваются разными цветами (Высокий — красный, Средний — оранжевый, Низкий — зеленый).
✔️ Строгая валидация
Использование INotifyDataErrorInfo. Название обязательно (от 3 симв.), а срок исполнения не может быть в прошлом.
💾 Асинхронный JSON
Загрузка при старте и автосохранение изменений в файл tasks.json без блокировки интерфейса (UI-потока).
🛠 Технологический стек
Framework: .NET 8 (WPF)
Архитектура: Чистый MVVM (без использования сторонних фреймворков типа Prism/MvvmLight).
Base Logic: Кастомные реализации BaseViewModel (с поддержкой валидации) и RelayCommand (синхронный/асинхронный).
Библиотеки:
Microsoft.Xaml.Behaviors.Wpf — для обработки событий (Loaded и др.) в XAML.
System.Text.Json — для сериализации данных.
UI Helpers: GridHelper для лаконичной разметки сеток и набор IValueConverter.
📦 Инструкция по запуску
Требования: Установленная Visual Studio 2022 и пакет .NET 8 SDK.
Клонирование:
bash
git clone [ССЫЛКА_НА_ВАШ_РЕПОЗИТОРИЙ]
Используйте код с осторожностью.

Сборка: Откройте .sln файл и выполните Build Solution (пакеты NuGet восстановятся автоматически).
Запуск: Нажмите F5. Файл данных tasks.json будет создан автоматически в директории приложения после добавления первой задачи.
📂 Структура проекта
Model/ — Сущности (TodoTask) и перечисления.
ViewModel/ — Контексты окон (TaskManagerContext, TaskEditContext).
View/ — XAML разметка окон и стили.
Services/ — Сервис для работы с файловой системой (TodoDataService).
Lib/ — Инфраструктурные классы (Команды, Хелперы, Базовые классы).
Converters/ — Конвертеры данных для визуального представления.
💡 Особенности реализации
Zero Code-Behind: Вся логика, включая закрытие окон и обработку событий загрузки, вынесена во ViewModel и XAML-триггеры.
Live Shaping: Использование ICollectionView позволяет выполнять фильтрацию и сортировку без физического изменения основной коллекции данных.
StatusBar: Нижняя панель информирует пользователя о статусе сохранения и общем количестве задач.
